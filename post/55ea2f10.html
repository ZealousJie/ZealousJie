记录我搭建dataease时遇到的问题

发表于2022-11-11|更新于2022-11-11|java
|阅读量:

一.前情提要

本人是根据公司前辈提供的**项目源码和一些其他的代码**以及根据官方的开发文档开始搭建的（为了大家的方便还是原封不动的贴出来吧），鄙人是菜逼，全文是我个人理解，说的不对地方希望你能指出来。

其他的情况：

  • 数据库使用的是MySQL8.0（这是个大坑）
  • IDEA2021.1.3版本
  • NavicatPremium 15
  • JDK8
  • 详细的环境配置请看开发文档

二.初步搭建

最好搭配官方开发文档一起食用

并且先看文档一步一步走（遇到问题可以再看看我这）

我们主要目标就是将我上面贴的压缩包里的项目代码跑起来，其中

  • dataease是最主要的项目代码
  • sdk可能是你编译项目时缺少的一些依赖
  • opt你可以把它当作这个项目的配置文件

然后开始我们的搭建

1.首先你最好将三个解压后的文件夹都放到D盘的根目录下

2.打开 D:\opt\dataease\conf\dataease.properties文件 修改配置

3.用idea分别打开dataease文件夹下面的项目和dataease-extension-sdk文件夹下代码

三.可能遇到的问题且开发文档没有提及的

如果前面的问题解决不了你的问题不妨把下面所有的问题都看一遍

1.编译运行dataease项目提示缺少依赖

使用maven-package dataease-extension-sdk将打包好的分别处于三个模块下的jar包放到D:\opt\dataease\plugins\default下，没有这个路径下的文件夹就自己创建！

2.编译运行dataease项目报错如下

3.mysql区分大小写的问题

怎么说呢这个问题，网上是说大小写的问题，但其实我遇到这个问题的时候不是大小写引起的，因为我是在windows环境下搭建，默认是不区分大小写的，但我也忘了怎么解决的了，不过应该不难

4.最关键的来了，编码不一致导致的问题

这个问题官方文档提出了，但是解决不了问题。。

原因应该是： 你是和我一样的用MySQL8.0（推荐使用5.6，因为5.7好像也有一个额外的问题）的大怨种

因为MySQL8.0默认编码是utf8mb4_0900_ai_ci,而这个项目给你创建的初始数据 表结构和字段的编码都是

和这个不一致的。这就需要你手动改每个表以及每个字段的编码，或者你创建dataease数据库时指定一个能和你表字段的编码一致的编码。

前一种方案是：

ALTER TABLE 表名 CONVERT TO CHARACTER SET utf8 COLLATE utf8_general_ci;

这样一个表一个表的改，遇到有外键的还要删了再建，遇到有索引的还要改表结构

后一种方案就是：

CREATE DATABASE dataease DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

这是开发文档让你做的事，我这里将utf8mb4全替换成了utf8了 就解决了，说实话我感觉我的解决方案都挺不正经的，下一个低版本的mysql少走这么多弯路。
